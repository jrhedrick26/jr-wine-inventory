# 🍇 JR Wine Cellar

A premium, modern, multi-user wine inventory and cellar tracking web application. The application is built using **Streamlit**, **Google Sheets** (acting as a secure relational database), and **Google Gemini** (providing advanced vision-based label scanning and conversational wine educational profiles).

---

## ✨ Features

* **📷 AI-Powered Wine Label Scanner**: Scan single or multiple labels concurrently. The application uses Google Gemini (with deterministic, zero-temperature vision settings) to extract the Winery, Varietal (including Cuvee selections and vineyard designations), and Vintage.
* **✍️ Smooth Manual Intake**: Prefills data dynamically from successful image scans, allowing simple additions and quantity counts.
* **🔐 Multi-User Separation Integrity**: Features a Gatekeeper login wall checking authorized users. All data remains completely isolated so users can only view, edit, or consume their own bottles, even though the database is stored in a single Google Sheet.
* **📊 Elegant Dashboard & Detail Panel**: High-end glassmorphic UI displaying key cellar stats (Total Active Bottles, Unique Varietals) and a detailed wine panel showing Origin, Tasting Notes, Pairings, Serving temperature tricks, and Fun Facts.
* **📋 Prepared Sharing Text**: Instantly copies a conversational summary of a bottle's detail profile to your clipboard to share with friends.
* **🍷 Consumption History**: Easily rate and log bottles as drank (with automated inventory decrementing or status transitioning), and restore consumed bottles back into the active cellar.
* **⚡ Optimized Session Cache**: Employs local dataframe caching to eliminate latency. Google Sheets API queries are only run upon login or immediately following a database write operation.

---

## 🛠️ Tech Stack

* **Frontend/Core**: Python, [Streamlit](https://streamlit.io/)
* **Database & Sheets API**: [gspread](https://github.com/burnash/gspread), Google Service Accounts
* **Data Processing**: Pandas, Pillow (PIL)
* **Generative AI**: [Google GenAI SDK](https://github.com/googleapis/python-genai) (utilizing Gemini Models)
* **Styling**: Vanilla CSS with HSL/Outfit typography in a customized glassmorphic dark theme

## 📋 Database Schema

The main inventory database relies on a structured sequence of columns inside the Google Sheets sheet:

| Column | Description |
| :--- | :--- |
| `user_code` | Unique access code representing the owner of the wine |
| `id` | Unique auto-incrementing integer key representing the bottle |
| `winery` | Producer or vineyard name |
| `varietal` | Grape variety or blend (with cuvée/vineyard designation appended in parentheses) |
| `vintage` | Production year (integer) |
| `status` | Current inventory state (`Active` or `Drank`) |
| `rating` | Consumption rating (`Loved`, `Liked`, `Disliked`, or `None`) |
| `wine_101` | Markdown educational profile generated by Gemini |
| `quantity` | Number of bottles in stock (integer) |

## 🚀 Setup & Installation

### 1. Prerequisites
Ensure you have Python 3.10+ installed.

### 2. Install Dependencies
Clone the repository, navigate to the folder, and install the required libraries:
```bash
pip install -r requirements.txt
```

### 3. Setup Streamlit Secrets
Create a `.streamlit/secrets.toml` file in the root directory and configure authorization secrets:

```toml
[auth]
gemini_api_key = "YOUR_GEMINI_API_KEY"
target_model = "gemini-flash-latest" # Fallback to gemini-flash-latest

[gcp_service_account]
type = "service_account"
project_id = "your-gcp-project"
private_key_id = "your-key-id"
private_key = "-----BEGIN PRIVATE KEY-----\n..."
client_email = "your-service-account@your-gcp-project.iam.gserviceaccount.com"
client_id = "..."
auth_uri = "https://accounts.google.com/o/oauth2/auth"
token_uri = "https://oauth2.googleapis.com/token"
auth_provider_x509_cert_url = "https://www.googleapis.com/oauth2/v1/certs"
client_x509_cert_url = "..."
universe_domain = "googleapis.com"
```

### 4. Running the App
Start the Streamlit development server locally:
```bash
streamlit run app.py
```
